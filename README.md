# 🧠 PDUA Workshop Toolkit

> **Language:** C, C++, PDUA Assembly  
> **Standard:** C11 / C++17  
> **Build System:** CMake + Ninja/Make  
> **Status:** Educational assembler + simulator for the **PDUA pedagogical processor**

A compact educational toolkit to:

- ✍️ write programs in **PDUA Assembly**
- 🏗️ assemble them into binary machine code
- ⚙️ simulate execution on the **PDUA processor**
- 🧪 compare results with equivalent **C reference programs**
- 🎨 use custom **VS Code syntax highlighting** for `.pdua` files

---

# 📚 Overview

This repository contains a small workflow for a pedagogical processor called **PDUA**. The project includes:

- a **PDUA assembler/compiler**
- a **PDUA simulator**
- optional **C reference programs** for result verification
- a **VS Code syntax extension** for `.pdua` files

It is designed for workshop and learning purposes, especially for:

- instruction encoding
- symbolic labels
- memory addressing
- loops and control flow
- arithmetic and bitwise operations
- arrays and simple low-level algorithms

---

# ✨ Features

## Assembler / Compiler

- 🔤 Parses `.pdua` source files
- 🏷️ Resolves labels and symbolic addresses
- 🧮 Supports immediate values in decimal and hexadecimal
- 📦 Generates:
  - `.bin` machine code
  - `.sym` symbol map

## Simulator

- 🧠 Simulates the PDUA processor state
- 📍 Shows registers and flags
- 🗂️ Displays memory maps
- 🏷️ Reads `.sym` files to show useful data labels and final values

## Instruction Set Extensions

The project also supports custom pedagogical instructions in the available opcode range:

- `XOR ACC, A`
- `SLL ACC, A`
- `SLR ACC, A`

## Editor Support

- 🎨 Custom syntax highlighting for `.pdua`
- 🧩 Optional VS Code extension project included in `pdua-syntax/`

# 🛠️ Build Configuration

This project uses:

- **C11** for reference C programs
- **C++17** for the assembler and simulator

The main CMake configuration is:

- `pdua_compiler` → assembles `.pdua` source into `.bin` and `.sym`
- `pdua_simulator` → simulates `.bin` programs
- `workshop_reference` → runs equivalent C reference logic

---

# 🚀 Build Process

## 1. Configure the project

```bash
cmake -B build -S .
```

## 2. Build all targets

```bash
cmake --build build
```

This generates:

- `build/pdua_compiler`
- `build/pdua_simulator`
- `build/workshop_reference`

---

# ▶️ Running Programs

## Assemble a PDUA source file

```bash
./build/pdua_compiler pdua_programs/pnt_1.pdua
```

This produces:

- `pdua_programs/pnt_1.bin`
- `pdua_programs/pnt_1.sym`

---

## Simulate the binary

```bash
./build/pdua_simulator pdua_programs/pnt_1.bin
```

---

## Run the C reference program

```bash
./build/workshop_reference
```

---

# ⚡ Helper Scripts

## Execute full PDUA flow

If you already have a helper script such as `execute_pdua_programs.sh`, run it like this:

```bash
./execute_pdua_programs.sh pdua_programs/pnt_1.pdua
```

This usually performs:

1. configure
2. build
3. assemble
4. simulate

> Avoid using `source execute_pdua_programs.sh ...` unless you intentionally want to run it inside the current shell.

# 🧾 Output Files

## `.bin`

Binary machine code produced by the assembler.

## `.sym`

Symbol map generated by the assembler, useful for:

- variable addresses
- data label locations
- debugging
- simulator-friendly output

Example:

```text
FOR_START 0x59
FOR_END 0xBD
VARN 0xBE
VART 0xBF
VAR1 0xC0
VAR2 0xC1
VARM 0xC2
VARI 0xC3
FIB0 0xC4
FIB1 0xC5
```

# 🏷️ Supported PDUA Syntax

## Labels

```asm
loop_start:
```

## Immediate values

```asm
0x14
10
```

## Comments

```asm
@ This is a comment
# Header-like comment
; Alternate comment style
```

## Data definitions

```asm
varN: 0x14
fib0: 0x00
fib1: 0x00
```

# 📌 Notes

- The PDUA processor is an **educational 8-bit architecture**
- Values may overflow modulo 256
- Array and loop logic are implemented under ISA constraints
- Some algorithms are intentionally verbose because the processor is minimal and pedagogical
